# multi-module-monolith-fixture

A scale-test fixture: an auto-generated multi-module Gradle project with
~150 trivial subprojects. Useful as a representative "large monolith"
workload when investigating IDE plugin behaviour at scale (indexing time,
memory pressure, file watcher behaviour, etc.).

## Shape

- **150** Gradle subprojects: `module-001` ... `module-150`
- **30** trivial Java classes per module (~60 LOC each) — total ~234K Java LOC
- **3** trivial TypeScript files per module
- A few `AGENTS.md` / `CLAUDE.md` files at root + a handful at module level
- Top-level `build.gradle.kts`, `settings.gradle.kts`, `gradle.properties`,
  `tsconfig.json`, `package.json`

There is **no real logic** in any of the generated files — they are
mechanically generated.

## Quick start

1. Clone this repository.
2. Open the cloned directory as a Gradle project in IntelliJ IDEA
   (any modern version).
3. Wait for Gradle import + indexing to fully complete.
4. Use as needed for your investigation.

## How this was generated

Mechanically generated by a script. The exact shape (module count, files
per module, etc.) is parameterised — see the sibling
[`multi-workspace-folder-fixture`](https://github.com/ghc-jetbrains/multi-workspace-folder-fixture)
for the IntelliJ-native variant whose generator script is committed.
